Include these terms (alphabetical order, per D5 + 06-RESEARCH.md term inventory):

1. **AIGF** (AI Governance Framework) — FINOS framework that auto-attaches governance overlays to architectures containing ai:* node types. Example: "The AIGF decorator set is automatically applied when a CALM doc contains an ai:agent node." Lesson: [Governance Frameworks and AaC](/docs/module-01-case-for-aac/governance-frameworks-and-aac)

2. **ARB** (Architectural Reference Board) — FINOS body that defines canonical architecture patterns used as CALM pattern sources. Example: "The FINOS Multi-Agent ARB pattern provides a validated blueprint for ai:agent orchestration." Lesson: [Governance Frameworks and AaC](/docs/module-01-case-for-aac/governance-frameworks-and-aac)

3. **Architecture as Code (AaC)** — The practice of expressing software architecture as versioned, machine-readable specification files rather than static diagrams. Example: "We store our system topology as a .calm.json file in git, alongside the code that implements it." Lesson: [Introducing CALM](/docs/module-01-case-for-aac/introducing-calm)

4. **CALM** (Common Architecture Language Model) — The FINOS open standard (current version: 1.2) for expressing software architecture as a machine-readable JSON schema. Example: "A valid CALM document passes calm validate -a." Lesson: [Introducing CALM](/docs/module-01-case-for-aac/introducing-calm)

5. **CALM Guard** — An AI-driven compliance automation system (6-agent squad) that validates CALM documents against governance controls. Example: "CALM Guard checks each architecture against CCC controls before it can be promoted." Lesson: [Governance Frameworks and AaC](/docs/module-01-case-for-aac/governance-frameworks-and-aac)

6. **CALM Hub** — The versioned registry of CALM artifacts (patterns, architectures, controls) — analogous to npm for architecture. Example: "The team publishes their approved architecture to CALM Hub as version 1.3.0." Lesson: [CALM Hub: Architecture Registry](/docs/module-03-calm-ecosystem/calm-hub-architecture-registry)

7. **CALM Server** — A headless service that exposes CALM Hub functionality over HTTP. Example: "The CI pipeline validates architecture changes against the team's CALM Server instance." Lesson: [CALM Server: Validation Service](/docs/module-03-calm-ecosystem/calm-server-validation-service)

8. **CALM Studio** — The browser-based visual canvas for CALM architecture authoring at studio.calm.finos.org. Example: "Use CALM Studio to drag nodes onto the canvas and generate the JSON automatically." Lesson: [CALM Studio: Visual Design](/docs/module-03-calm-ecosystem/calm-studio-visual-design)

9. **calmstudio-mcp** — A Model Context Protocol server that enables AI assistants (Claude, Copilot) to create and validate CALM documents directly. Example: "With calmstudio-mcp active, you describe your system in 3 sentences and Claude generates the .calm.json." Lesson: [Three Paths to Your First CALM Document](/docs/module-00-on-ramp/three-paths-to-first-calm-doc)

10. **control** — A governance constraint attached to a CALM architecture node or relationship, mapping to a compliance framework requirement. Example: "The database node has a control referencing CCC-DATA-003 requiring encryption at rest." Lesson: [Controls](/docs/module-02-calm-fundamentals/controls)

11. **decorator** — A CALM metadata extension that adds cross-cutting concerns (governance, compliance, AI attributes) to nodes or relationships without changing the structural schema. Example: "An ai:* decorator on a service node signals it contains model inference logic subject to AIGF." Lesson: [Decorators](/docs/module-02-calm-fundamentals/decorators)

12. **FINOS CCC** (Common Cloud Controls) — The FINOS catalog of cloud control requirements used as the standard control vocabulary in CALM governance. Example: "A CALM Guard validation run maps each control to a CCC identifier." Lesson: [Governance Frameworks and AaC](/docs/module-01-case-for-aac/governance-frameworks-and-aac)

13. **Gemara** — The OpenSSF 7-layer GRC Engineering Model. CALM is its Layer 4 (Sensitive Activities). Example: "The Gemara stack sits below the team's development workflow — CALM is the layer where architectural risk is captured." Lesson: [Governance Frameworks and AaC](/docs/module-01-case-for-aac/governance-frameworks-and-aac)

14. **GRIS** (Generative Regulatory Intelligence System) — A FINOS tool that maps regulatory text to CALM controls. Example: "GRIS parsed the MiFID II text and generated the corresponding CALM control set." Lesson: [Governance Frameworks and AaC](/docs/module-01-case-for-aac/governance-frameworks-and-aac)

15. **interface** — A CALM schema object that defines the transport and protocol contract between two connected nodes. Example: "The HTTPS interface specifies TLS 1.3 and port 443 between the webclient and service nodes." Lesson: [Interfaces](/docs/module-02-calm-fundamentals/interfaces)

16. **node type** — One of the 9 core CALM types that classify architectural components: actor, ecosystem, system, service, database, network, ldap, webclient, data-asset. Example: "A PostgreSQL database maps to the database node type; a React SPA maps to webclient." Lesson: [Nodes](/docs/module-02-calm-fundamentals/nodes)

17. **pattern (CALM)** — A reusable, parameterized CALM architecture template stored in CALM Hub and instantiated for specific use cases. Example: "The team forked the FINOS microservices pattern and set their own service names." Lesson: [Patterns and Standards](/docs/module-03-calm-ecosystem/patterns-and-standards)

18. **relationship** — A directed CALM link between two nodes representing a data flow, dependency, or communication path. Example: "A connects relationship from the webclient node to the api-gateway service documents the REST API call flow." Lesson: [Relationships](/docs/module-02-calm-fundamentals/relationships)

19. **SAIF** (Secure AI Framework) — Google's security framework for AI systems, mappable to CALM ai:* node types and decorators. Example: "The SAIF threat surface map is used to choose which ai:* decorators apply to each model node." Lesson: [Governance Frameworks and AaC](/docs/module-01-case-for-aac/governance-frameworks-and-aac)

20. **validation** — The process of checking a CALM JSON file against the 1.2 schema using calm validate -a. Example: "calm validate -a my-system.calm.json exits 0 if the file is schema-valid." Lesson: [The CALM CLI: Your Architecture Toolbox](/docs/module-03-calm-ecosystem/cli-toolbox)
