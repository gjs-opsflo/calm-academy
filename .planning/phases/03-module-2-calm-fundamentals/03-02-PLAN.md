CALM 1.2 interface forms (from /Users/gshah/work/opsflow-sh/calm/architecture-as-code/calm/release/1.2/meta/interface.json — verified by Wave 0 research):

Form 1 — interface-definition (formal, definition-url + config):
```
{
  "unique-id": "string",
  "definition-url": "URL to external JSON Schema",
  "config": { ...satisfies that schema... }
}
```

Form 2 — interface-type (freeform):
```
{
  "unique-id": "string",
  "name": "...",
  "protocol": "HTTPS",
  "port": 8080,
  "...": "any additional properties"
}
```

Interfaces live in a node's `interfaces[]` array. Relationships reference them via `source.interfaces[]` and `destination.interfaces[]` (string id arrays).

Control shape (verified from control.json):
```
"controls": {
  "<control-key>": {
    "description": "string",
    "requirements": [
      { "requirement-url": "URI", "config": { ... } }
    ]
  }
}
```
Control key matches `^[a-zA-Z0-9-]+$`. Each requirement needs `requirement-url` (required) and either `config` (inline) or `config-url` (external) — oneOf. Controls attach at top-level architecture, per-node, per-relationship, or per-flow.

Decorator document shape (verified from decorators.json) — SEPARATE file, not inside architecture:
```
{
  "unique-id": "string",
  "type": "free-form string",
  "target": ["architecture-file-path-or-url", "..."],
  "applies-to": ["node-or-relationship-unique-id", "..."],
  "data": { "minProperties": 1 — free-form object }
}
```
