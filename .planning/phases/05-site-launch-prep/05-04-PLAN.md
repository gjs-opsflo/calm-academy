## Phase Goal

**As a** prospective FINOS reviewer or new learner, **I want to** open https://gjs-opsflo.github.io/calm-academy/ in my browser and read any chapter or take any quiz from Modules 0–3, **so that** I can evaluate the course without needing to clone the repo or run anything locally.

<objective>
Ship the public deployment vertical slice: a single GitHub Actions workflow at `.github/workflows/site-deploy.yml` that runs `npm ci` to install dependencies (including `js-yaml` from devDependencies), then runs the prebuild YAML→JSON conversion, builds the Docusaurus site, configures Pages context (`actions/configure-pages@v4` per D7), uploads the build artifact, and deploys to GitHub Pages via the native Pages workflow (per CONTEXT.md D7 and D10). The workflow runs on push to `main` (deploys) and on pull_request (build-only verification). After this plan, every push to `main` produces an updated live site at `https://gjs-opsflo.github.io/calm-academy/` with all 4 modules and all 4 quizzes accessible.

Purpose: Local builds pass after Plans 01–03, but SITE-01 requires a deployed production-quality site. A static export is not enough — the FINOS demo and 10 internal beta testers (ROADMAP Phase 5 Success Criterion #7) need a stable URL. The native Pages workflow (NOT `gh-pages` npm or `peaceiris/actions-gh-pages`, per CONTEXT.md D7) is the officially-supported pattern and matches the existing `ci.yml` style (Node 22, actions/checkout@v4, npm cache scoped to `site/package-lock.json`). Output: One new workflow file. After it lands and runs, the live URL serves the site.

This is a blocking-human checkpoint — Claude cannot toggle repo settings via the GitHub web UI without elevated credentials.

<!-- Complete workflow shape — CORRECTED step order: npm ci runs BEFORE convert-quiz-yaml.mjs to ensure js-yaml is installed first -->

Key workflow facts:
- name: `Deploy to GitHub Pages`
- triggers: `push` to `main` (deploys) AND `pull_request` to `main` (build-only)
- permissions: `contents: read`, `pages: write`, `id-token: write` (required for OIDC token used by deploy-pages action)
- concurrency: group `pages`, `cancel-in-progress: false` (prevents simultaneous deploys but does not cancel in-progress runs — Pages best practice)
- jobs: `build` (always runs) + `deploy` (needs build, only on push to main via `if: github.ref == 'refs/heads/main'`)

Build job steps (in order — CRITICAL: npm ci must run BEFORE convert-quiz-yaml.mjs because js-yaml is in site/package.json devDependencies and is not installed until npm ci runs):
1. `actions/checkout@v4` with `fetch-depth: 1` (shallow clone — full history is unnecessary for a static site build; `fetch-depth: 0` would download every commit and slow CI for no benefit)
2. `actions/setup-node@v4` with `node-version: 22`, `cache: npm`, `cache-dependency-path: site/package-lock.json`
3. Run: `cd site && npm ci` (MUST run BEFORE the quiz prebuild step — this installs js-yaml which the next step requires; named "Install site dependencies")
4. Run: `node scripts/convert-quiz-yaml.mjs` (the prebuild step; converts quizzes/*.yaml → site/src/quizzes/*.json — js-yaml is now available because step 3 ran first; named "Convert quiz YAML to JSON")
5. Run: `cd site && npm run build` (named "Build site")
6. (conditional on main) `actions/configure-pages@v4` (named "Configure GitHub Pages") with `if: github.ref == 'refs/heads/main'` — sets up the Pages environment context and baseUrl injection; MUST come BEFORE upload-pages-artifact per D7's locked action sequence
7. (conditional on main) `actions/upload-pages-artifact@v4` with `path: site/build` AND `if: github.ref == 'refs/heads/main'`

Deploy job:
- `needs: build`, `if: github.ref == 'refs/heads/main'`
- `environment: { name: github-pages, url: ${{ steps.deployment.outputs.page_url }} }`
- step id `deployment` runs `actions/deploy-pages@v4`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner ← repo (push event) | Untrusted PR code could execute in runner context (but build job has read-only token; deploy gated to main) |
| GitHub Pages CDN ← Actions artifact | Pages serves whatever artifact we upload; supply chain integrity matters |
| GitHub OIDC token → deploy-pages | Token has `pages: write`; must not be exposed to untrusted code |
| Live site ← public internet | Public read-only; no auth, no user data |
