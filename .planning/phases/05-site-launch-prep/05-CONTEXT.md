# Phase 5 — Site Launch Prep: Implementation Context

**Phase:** 05-site-launch-prep
**Date captured:** 2026-06-16
**Status:** Ready for planning

---

## Phase Goal

Scaffold and deploy the Docusaurus site so CALM Academy content is publicly browsable at a stable URL. Wire existing content (lessons, illustrations, quizzes) into the site. Ship a working, searchable site with CI/CD auto-deploy.

---

## Locked Decisions

### D1 — Content Wiring Strategy

**Decision:** Symlink `content/` into `site/docs/`

- `site/docs/` → symlink → `content/` (or subdirectory per module)
- Illustrations: symlink `illustrations/exported/` → `site/static/img/`
- Docusaurus resolves relative image paths from `static/img/` at build time
- No file copying — single source of truth stays in `content/`
- Symlinks tracked in git (Unix; CI runner is Linux — no Windows concern)

### D2 — Sidebar Navigation

**Decision:** Autogenerated sidebar from directory structure

- `sidebars.js` uses `{ type: 'autogenerated', dirName: '.' }`
- Module directories already kebab-cased (`module-00-on-ramp/`, `module-01-why-aac/`, etc.)
- Explicit ordering via numeric prefix on filenames (already in place: `00-`, `01-`, etc.)
- No hand-maintained sidebar config — auto updates as content grows

### D3 — Quiz Component Architecture

**Decision:** Custom React MDX component `<Quiz />`

- Component: `site/src/components/Quiz.tsx`
- Usage in MDX: `<Quiz file="module-03-ecosystem.yaml" />`
- Quiz YAML files read at build time (Docusaurus plugin or import) and bundled
- Grading: submit-all (one submit button for the whole quiz, not per-question)
- On submit: show score + per-question explanations
- `short_answer` type: rendered as read-only — model answer revealed on submit (no freetext grading)

### D4 — Quiz Data Loading

**Decision:** Static import via Webpack/Docusaurus loader

- YAML files loaded via `js-yaml` at build time
- Quiz component receives parsed JS object (not raw YAML string)
- No server-side API — fully static, no backend needed for launch
- `quizzes/` directory stays at repo root; site build copies/imports from there

### D5 — Docusaurus Theme

**Decision:** Classic theme, minimal customization for launch

- `@docusaurus/preset-classic` — no custom theme package
- Color: default Docusaurus blue; CALM/FINOS brand colors deferred to post-MVP
- Logo: placeholder (text "CALM Academy") — real logo deferred
- Navbar: Home, Docs, (GitHub link)
- Footer: minimal — FINOS mention, Apache 2.0 / CC BY-SA 4.0 note

### D6 — Search

**Decision:** Algolia DocSearch

- `@docusaurus/plugin-search-algolia` (built-in, no extra install)
- Apply for DocSearch crawl after first public deploy
- Placeholder Algolia config in `docusaurus.config.js` until approved
- Fallback: local search plugin (`@easyops-cn/docusaurus-search-local`) during dev

### D7 — Deploy Target

**Decision:** GitHub Pages via GitHub Actions native Pages workflow

- Actions: `actions/configure-pages` + `actions/upload-pages-artifact` + `actions/deploy-pages`
- **Not** `gh-pages` npm package or `peaceiris/actions-gh-pages`
- Branch: Pages deploys from GitHub Actions artifact (not a `gh-pages` branch)
- Trigger: push to `main`

### D8 — Site URL

**Decision:** `https://gjs-opsflo.github.io/calm-academy/`

- `url`: `https://gjs-opsflo.github.io`
- `baseUrl`: `/calm-academy/`
- All internal links must use `baseUrl`-relative paths (Docusaurus handles this automatically)
- Custom domain deferred until FINOS transfer

### D9 — Site Directory Structure

**Decision:** `site/` subdirectory at repo root

```
calm-academy/
├── content/          ← lesson MDX (source of truth)
├── illustrations/    ← Excalidraw + exported SVG
├── quizzes/          ← YAML quiz files
├── labs/             ← lab content
├── code-examples/    ← CALM JSON
├── slides/           ← Marp decks
└── site/             ← Docusaurus project
    ├── docs/         ← symlink → ../content/
    ├── static/
    │   └── img/      ← symlink → ../../illustrations/exported/
    ├── src/
    │   └── components/
    │       └── Quiz.tsx
    ├── docusaurus.config.js
    ├── sidebars.js
    └── package.json
```

### D10 — CI Pipeline

**Decision:** Single GitHub Actions workflow for build + deploy

- File: `.github/workflows/site-deploy.yml`
- Steps: checkout → setup-node → install deps → build → configure-pages → upload artifact → deploy
- Node version: 20 (LTS)
- Runs on: `push` to `main`, `pull_request` (build-only, no deploy on PR)
- Separate job: `validate-calm` (existing or new) — `npx @finos/calm-cli validate` on all `*.calm.json`
