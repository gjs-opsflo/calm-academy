**Node.js equivalent pattern** (from `05-RESEARCH.md` lines 394–414 — use this structure, it's a `.mjs` not `.sh`):
```javascript
// scripts/convert-quiz-yaml.mjs
// Run: node scripts/convert-quiz-yaml.mjs (before npm run build in CI)
import { readFileSync, writeFileSync, mkdirSync, readdirSync } from 'fs';
import { load } from 'js-yaml';
import { join } from 'path';

const quizDir = 'quizzes';
const outputDir = 'site/src/quizzes';

mkdirSync(outputDir, { recursive: true });

for (const file of readdirSync(quizDir)) {
  if (!file.endsWith('.yaml')) continue;
  const data = load(readFileSync(join(quizDir, file), 'utf8'));
  writeFileSync(
    join(outputDir, file.replace('.yaml', '.json')),
    JSON.stringify(data, null, 2)
  );
}
console.log('Quiz YAML → JSON conversion complete.');
```

| Input YAML | Output JSON (imported by MDX) |
|-----------|-------------------------------|
| `quizzes/module-00-on-ramp.yaml` | `site/src/quizzes/module-00-on-ramp.json` |
| `quizzes/module-01-case-for-aac.yaml` | `site/src/quizzes/module-01-case-for-aac.json` |
| `quizzes/module-02-calm-fundamentals.yaml` | `site/src/quizzes/module-02-calm-fundamentals.json` |
| `quizzes/module-03-ecosystem.yaml` | `site/src/quizzes/module-03-ecosystem.json` |

```yaml
# .github/workflows/site-deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    name: Build Docusaurus
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: npm
          cache-dependency-path: site/package-lock.json
      - name: Convert quiz YAML to JSON
        run: node scripts/convert-quiz-yaml.mjs
      - name: Install site dependencies
        run: cd site && npm ci
      - name: Build site
        run: cd site && npm run build
      - name: Upload Pages artifact
        if: github.ref == 'refs/heads/main'
        uses: actions/upload-pages-artifact@v3
        with:
          path: site/build

  deploy:
    name: Deploy to GitHub Pages
    needs: build
    if: github.ref == 'refs/heads/main'
    permissions:
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

**Key difference from `ci.yml`:** Site-deploy uses `actions/configure-pages` + `actions/upload-pages-artifact` + `actions/deploy-pages` (native Pages workflow). `ci.yml` only builds, does not deploy. Both coexist — `site-deploy.yml` is additive.

### Image path fixes in `content/**/*.mdx` (content modification, 21 files, 35 refs)

**Analog:** Existing MDX files are the analog — this is a surgical string replacement within them.

**Current broken patterns** (verified by `rg` across all modules):

Module-00 and module-01 (wrong depth):
```markdown

```

Module-02 and module-03 (wrong depth AND wrong for Docusaurus):
```markdown

```

**Correct Docusaurus pattern** — absolute path from `static/` (from `05-RESEARCH.md` lines 506–519):
```markdown
![Alt text](/img/m00-three-paths.svg)
```

**Replacement rule:** Extract the filename stem (basename) from any path matching `**/illustrations/exported/<filename>.svg`, replace the entire relative path with `/img/<filename>.svg`.

**Script approach** (from `05-RESEARCH.md` lines 693–699):
```bash
# Run from repo root — fixes both ../../ and ../ depth variants
find content/ -name "*.mdx" -exec sed -i '' \
  's|[.][.]/\([.][.]/\)*illustrations/exported/\([^)]*\)|/img/\2|g' {} \;
```

**Verification after fix:**
```bash

```
