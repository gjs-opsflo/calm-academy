## Phase Goal

**As a** learner who landed on the docs site after Plan 01, **I want to** see all 24 chapters across Modules 0–3 listed in the sidebar in correct pedagogical order, **so that** I can navigate the course from start to finish without sorting the chapters in my head.

<objective>
Deliver the navigation vertical slice: add the `sidebar_position` frontmatter field to all 24 existing MDX files (deriving the value from the existing `chapter: M.N` field, where N becomes the position), and add 4 new `_category_.json` files (one per module directory) that label and order the module sections in the sidebar. After this plan, the autogenerated Docusaurus sidebar (configured in Plan 01) reflects the canonical curriculum order: Module 0 → 1 → 2 → 3, with chapters 0.1 → 0.5, 1.1 → 1.5, 2.1 → 2.7, 3.1 → 3.7 in each section.

Purpose: Without sidebar ordering metadata, Docusaurus sorts entries alphabetically by filename — so `building-your-first-architecture.mdx` (chapter 2.7, the FINAL chapter of Module 2) sorts BEFORE `calm-specification.mdx` (chapter 2.1, the FIRST chapter). This is pedagogically broken. The fix is metadata-only — no content changes — but it must be precise.

Output: 4 new `_category_.json` files + 24 MDX files with one new frontmatter field each. Sidebar renders in pedagogical order after `npm run build`.
</objective>
