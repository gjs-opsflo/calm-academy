## Phase Goal

**As a** learner who finished reading Module 0, **I want to** take an embedded quiz at the end of the final chapter, submit my answers, and see my score with explanations, **so that** I can verify my understanding before moving on to Module 1 — without leaving the site, without creating an account, and without a backend.

<objective>
Deliver the quiz-taking vertical slice end-to-end under strict TDD discipline: build the prebuild YAML→JSON converter script, install vitest and write 15 failing tests that codify the UI-SPEC behavioral contract, then implement `Quiz.tsx` to make those 15 tests pass (no implementation code before tests are red — CLAUDE.md TDD rule applies unconditionally), and wire the verified component into the last chapter of each of the 4 modules (Module 0 → `your-first-calm-document.mdx` chapter 0.5; Module 1 → `governance-frameworks-and-aac.mdx` chapter 1.5; Module 2 → `building-your-first-architecture.mdx` chapter 2.7; Module 3 → `cicd-integration.mdx` chapter 3.7). Prove the slice works by running the build, then in dev mode answering, submitting, and observing the score for at least one quiz.

Purpose: Quizzes are the assessment surface for learners. CONTEXT.md D3 specifies a single React MDX component (`<Quiz />`) used identically across all 4 modules, submit-all grading, and read-only short-answer with model answers revealed post-submit. This plan delivers ALL four quizzes (not a "v1 with one quiz" subset) because: (a) UI-SPEC requires production-quality interaction; (b) wiring 3 more imports + 3 more `<Quiz>` tags is trivial once the component exists; (c) the requirement SITE-01 says "quizzes rendering via custom MDX component" — plural — and we never split a locked decision into "v1/v2" per the planner scope reduction prohibition. Per the checker revision: TDD is enforced — Task 2 produces 15 failing tests before Task 3 produces any implementation; no escape hatch to "manual verification".

Output: 1 prebuild script (`scripts/convert-quiz-yaml.mjs`), 1 vitest config (`site/vitest.config.ts`), 1 test file with 15 failing tests (`site/src/components/Quiz.test.tsx`), 1 React component (`site/src/components/Quiz.tsx`) that turns those tests green, 1 `.gitignore` for the generated JSON output dir, an updated `site/package.json` with vitest devDependencies and a `prebuild` script hook, and 4 MDX files with embedded `<Quiz />` invocations. After this plan, all 4 quizzes are taking-ready locally and the component has a 15-test regression harness.
</objective>
