# Phase 5 — UI Design Contract: Site Launch Prep

> Visual and interaction contract for the Docusaurus site scaffold and Quiz MDX component.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

This phase produces a **documentation site** (Docusaurus), not a product UI. The design contract is scoped to:
1. `Quiz.tsx` — the only custom React component authored in this phase
2. Docusaurus classic theme configuration — navbar, footer, minimal overrides
3. Site structure — no custom page layouts beyond Docusaurus defaults

All Docusaurus internals (sidebar, search bar, doc pages, code blocks, admonitions) use the
default `@docusaurus/preset-classic` theme unchanged. Do not re-specify them here.

## Spacing Scale

Quiz component spacing uses the 8-point scale to match Docusaurus body rhythm.
Do not introduce custom CSS variables — use inline values or `style` props scoped to `Quiz.tsx`.

| Token | Value | Usage in Quiz component |
|-------|-------|-------------------------|
| xs | 4px | Option bullet gap, explanation icon padding |
| sm | 8px | Spacing between option label and radio input |
| md | 16px | Padding inside question card, gap between question prompt and options |
| lg | 24px | Gap between questions within a chapter section |
| xl | 32px | Gap between chapter sections within the quiz |
| 2xl | 48px | Margin above the Submit button |
| 3xl | 64px | Outer padding of the quiz container (top/bottom) |

Exceptions:
- Touch targets for radio inputs and submit button: minimum 44px height (accessibility requirement — overrides 8pt scale where needed)
- Score display badge: 40px height (intentionally off-scale to read as a badge, not body text)

## Color

Docusaurus classic default palette. No custom colors for launch.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff (light) / #1b1b1d (dark) | Page background, quiz container background |
| Secondary (30%) | #f6f7f8 (light) / #2a2a2d (dark) | Question card background, chapter section dividers |
| Accent (10%) | Docusaurus default blue `#2e8555` (primary) | Submit button, correct answer highlight border |
| Destructive | Docusaurus default red `#fa383e` | Incorrect answer highlight border only |

Accent reserved for: Submit button background, correct answer card left border (on submit only).
Do NOT use accent on option hover, focus rings, or any unanswered state. Use browser default focus outline for keyboard navigation.

Correct/incorrect state colors apply ONLY after the user clicks Submit. Before submit, all options use the secondary background with no color differentiation.

Note: Docusaurus ships dark mode by default. Quiz component must work in both light and dark modes using CSS custom properties already provided by `@docusaurus/preset-classic` (`--ifm-color-primary`, `--ifm-background-color`, `--ifm-card-background-color`). Do not hardcode hex values in `Quiz.tsx` — reference `--ifm-*` variables.

## Quiz Component: Interaction States

This is the core design contract for `Quiz.tsx`. Define every state precisely.

### States

| State | Trigger | Visual Appearance |
|-------|---------|-------------------|
| Unanswered | Initial render | All option rows: secondary background, no border color, radio unchecked. Submit button: disabled (opacity 0.5, cursor not-allowed). "Select an answer" placeholder hint shown below each question prompt. |
| Answered (pre-submit) | User selects a radio option | Selected option row: left border 2px solid `--ifm-color-primary`. All other options: unchanged. Submit button: enabled (full opacity, pointer cursor). Hint text hidden. |
| Partially answered (multiple questions) | Some questions answered, some not | Submit button remains enabled. Unanswered questions show hint text "Please answer this question to see your score" injected after submit attempt. |
| Submitted | User clicks Submit | All options become read-only (pointer-events: none). Score badge appears. Per-question result (correct/incorrect) and explanation text revealed. |
| Correct (post-submit) | Submitted, selected option has `correct: true` | Correct option row: left border 2px solid `--ifm-color-success` (#2e8555), light green background tint. "Correct" label badge (14px, weight 600, green) above explanation. |
| Incorrect (post-submit) | Submitted, selected option has `correct: false` | Selected option row: left border 2px solid `--ifm-color-danger` (#fa383e), light red background tint. "Incorrect" label badge (14px, weight 600, red). Correct answer row highlighted with green border (even though not selected). |
| Short answer (pre-submit) | Question type is `short_answer` | Textarea rendered read-only with placeholder: "Your answer (model answer revealed after you submit the full quiz)". No freetext input or grading. |
| Short answer (post-submit) | Submitted, type is `short_answer` | Model answer text block revealed below the textarea. Label: "Model Answer" (14px, weight 600). Textarea border: secondary (no correct/incorrect color — not graded). |
| Error: quiz load failure | `data` prop is undefined or malformed | Error state block rendered in place of quiz. (See Copywriting Contract for error copy.) |

### Multiple correct (`multiple_correct` type)

| Aspect | Specification |
|--------|---------------|
| Input type | Checkboxes, not radios |
| Submit enablement | Submit enabled after at least one checkbox checked |
| Post-submit scoring | All selected options individually marked correct/incorrect. "You got N of M correct answers" sub-label shown below score for `multiple_correct` questions. |

### Submit button behavior

- Disabled state: before any question is answered AND on first render
- Enabled state: once at least one question has any answer
- Label before submit: "Submit Quiz" (see Copywriting Contract)
- After submit: button replaced by "Retake Quiz" button. Retake clears all answers and returns to unanswered state.
- Do NOT show submit button again after submission — replace it inline.

## Accessibility

These constraints apply to `Quiz.tsx` only. Docusaurus handles the rest.

| Requirement | Specification |
|-------------|---------------|
| Radio group semantics | Each question's options wrapped in `<fieldset>` with `<legend>` containing the question prompt text |
| Checkbox group semantics | Same as radio — `<fieldset>` + `<legend>` for `multiple_correct` questions |
| Label association | Every `<input type="radio">` and `<input type="checkbox">` has a matching `<label>` via `htmlFor` / `id` pair. No click-area dependency on visual proximity alone. |
| Focus ring | Do not suppress the browser default focus ring. `outline: none` is forbidden in `Quiz.tsx`. |
| Touch target | Radio/checkbox click area: minimum 44px × 44px. Extend via padding on the label element, not the input. |
| Submit button | `<button type="button">`, not `<input type="submit">`. Disabled state must use `disabled` attribute (not just visual styling) so screen readers announce it. |
| Score announcement | Wrap score badge in `role="status"` `aria-live="polite"` so screen readers announce the result after submit without requiring focus. |
| Error announcement | Wrap error block in `role="alert"` so screen reader announces on quiz load failure. |
| Color not sole indicator | Correct/incorrect state must combine color + label text badge ("Correct" / "Incorrect"). Color alone is insufficient. |
| Keyboard navigation | Tab through questions → options → submit in DOM order. Arrow keys navigate within a radio group (native browser behavior — no custom key handlers needed). |
| `short_answer` textarea | `readOnly` attribute set. `aria-label="Your answer (read only — model answer revealed after submit)"`. |

WCAG level: 2.1 AA minimum. Phase 6 will audit; Phase 5 must implement these contracts.

## Mobile Responsiveness

Docusaurus classic theme is mobile-responsive by default. Quiz component constraints:

| Breakpoint | Behavior |
|------------|----------|
| < 480px | Quiz container: full width, 0 horizontal padding (let Docusaurus page padding apply). Option labels wrap — do not truncate. Submit button: full width. Score badge: full width, centered. |
| 480px – 768px | Quiz container: padded to md (16px) on each side. Submit button: auto width, left-aligned. |
| > 768px | Quiz container: max-width 760px, centered within doc page content area. Same max-width as Docusaurus code blocks. |

Chapter section headings collapse naturally — no horizontal scroll at any breakpoint.
