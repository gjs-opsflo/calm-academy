> Per-phase validation contract for feedback sampling during execution.

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | Docusaurus built-in build check (`onBrokenLinks: 'throw'`) + vitest (Quiz.tsx unit tests) |
| **Config file** | `site/docusaurus.config.js` (Wave 1); `site/vitest.config.ts` (Wave 3) |
| **Quick run command** | `cd site && npm run build 2>&1 \| tail -30` |
| **Full suite command** | `cd site && npm run build && npx vitest run` |
| **Estimated runtime** | ~60–90 seconds (build) + ~5 seconds (unit tests) |

---

## Sampling Rate

- **After every task commit:** `cd site && npm run build` (catches broken links, missing imports, TS errors)
- **After every plan wave:** Full build + visual check in browser at `localhost:3000` + verify all relevant pages render
- **Before `/gsd:verify-work`:** Full suite must be green (`npm run build` + `npx vitest run`)
- **Max feedback latency:** ~90 seconds
